# JINGLE_PROMPTS_2.md — per-program theme openers (Suno production brief, batch 2)

> Companion to `docs/JINGLE_PROMPTS.md`. That pack built the station's core identity + the
> 4 old daypart themes; the grid has since grown to ~28 named programs (`docs/programming/grid.yaml`),
> so **every program now gets its own opening theme**. This doc is the deliverables list for those.
>
> **Read `JINGLE_PROMPTS.md` §0 (the brand sound) and §1 (Suno mechanics) first** — the palette,
> the shared 3–5-note **glass-bell motif**, the three energy tiers, and the default **Exclude Styles**
> set all still apply. Every style below is **instrumental** (toggle Instrumental ON), carries the
> motif, and is self-contained. Generate 2–4 takes, keep the best, **trim to ~8–12 s, fade the tail
> clean**, and save the MP3 at the exact path in the first column.
>
> **The filename is load-bearing:** it is the program id from the grid. The code resolves a program's
> theme by that convention (`assets/themes/<program_id>.mp3`), so the name *is* the wiring — see
> `docs/PHASE_D_JINGLES_TASKS.md`. Nothing is discovered by scanning; a missing file just means that
> program opens on its format theme instead, never a crash.
